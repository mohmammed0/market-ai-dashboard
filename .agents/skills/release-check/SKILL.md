# release-check

## Description
Use this skill before handing off significant repo changes.

## When To Use
- Final verification
- Release-readiness checks
- Multi-page or multi-route changes

## Rules
- Separate fully verified, partially verified, scaffolded, and blocked items.
- Include exact PowerShell commands for backend, frontend, desktop UI, and direct API checks.
- Call out environment/runtime blockers explicitly.
- Do not claim full verification if tooling or network limits prevented it.
